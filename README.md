# IKHTARNI — Your Path to Smarter Hiring 🎯

> An AI-powered MERN recruitment platform that uses BERT-based NLP to automatically score candidate–job compatibility, helping recruiters cut through high application volumes and helping candidates apply with confidence.

---

## 📌 Overview

**IKHTARNI** is a full-stack predictive recruitment platform built as a PFA internship project during my first year at ENSIAS, my first hands-on experience combining full-stack engineering with applied AI/NLP. It addresses a real bottleneck in modern hiring: companies are flooded with applications, and manually screening every CV against every job posting is slow, expensive, and inconsistent.

IKHTARNI automates that first filtering step. Candidates upload a CV once and apply to multiple postings; recruiters get an instant **compatibility score** for every applicant, generated by a BERT-based NLP pipeline that compares CV content against job requirements across education, experience, and skills.

The platform was designed, developed, tested, and deployed end-to-end (frontend, backend, AI scoring engine, CI/CD, cloud hosting), solo.

---

## ✨ Key Features

### For Candidates
- Secure sign-up / login (email or Google) via **Clerk**
- Browse and filter job postings by **location** and **category**
- Upload, view, and update a CV (PDF) — required before applying
- Apply to postings, with duplicate-application protection ("Already applied")
- Track application status (Accepted / Pending / Rejected) from a personal dashboard

### For Recruiters / Companies
- Company account creation & secure login
- Post job openings (title, description, location, category, seniority level, salary)
- View all applicants per posting with their **AI-generated compatibility score**
- Accept or reject applications, or defer the decision
- Manage and track all active job postings from a recruiter dashboard

### For Visitors
- Browse the platform and open job listings without creating an account
- Filter postings by location and category

## 🧠 The AI Scoring Engine

The core differentiator of IKHTARNI is its NLP-driven scoring pipeline, run as a separate Python service:

**1. CV Information Extraction**
PDF resumes are parsed into structured text using **Apache Tika**, **PyPDF2**, and **pdfminer**, then processed with **Regex** + **spaCy** to extract:
- Candidate identity (name, email, phone)
- Technical skills (keyword detection: Python, Django, React, etc.)
- Education (degrees, years, institutions)
- Work experience (companies, durations)

**2. Semantic Scoring with BERT**
A pre-trained **`bert-base-uncased`** model converts both the CV and the job description into embeddings, then compares them across three weighted criteria:

| Criterion    | Weight | What it measures |
|--------------|:------:|-------------------|
| Education    | 2      | Candidate's degree(s) vs. job requirements |
| Experience   | 5      | Years of relevant experience vs. job requirements |
| Skills       | 3      | Cosine similarity between extracted skills and job-required skills |

The weighted result is returned as a **compatibility percentage**, giving recruiters an objective, ranked view of every applicant.

**3. Integration**
The Python scoring service exposes a `/receive_text` endpoint consumed by a dedicated Node.js server (`dummyserver.js`), which stores results in MongoDB and surfaces them on the recruiter dashboard in real time.

> ⚠️ **Known limitation:** BERT is loaded in memory (~420MB + runtime overhead), so scoring can fail on low-resource machines with insufficient virtual memory — a tradeoff documented and tested during the project.

## 🏗️ Architecture

```
┌─────────────┐        ┌──────────────────┐        ┌────────────────────┐
│  Frontend   │ <────> │   Backend / API   │ <────> │     MongoDB         │
│  (React +   │  REST  │  server.js        │        │  users / companies  │
│  Tailwind)  │        │  (Node + Express) │        │  jobs / applications│
└─────────────┘        └─────────┬─────────┘        └────────────────────┘
                                  │
                                  │ /receive_text
                                  ▼
                        ┌──────────────────┐
                        │  dummyserver.js   │
                        │  (scoring proxy)  │
                        └─────────┬─────────┘
                                  │
                                  ▼
                        ┌──────────────────┐
                        │  resume_matcher   │
                        │  (Python + BERT   │
                        │   NLP pipeline)   │
                        └──────────────────┘
```

The backend runs as two cooperating services: `server.js` for core app logic (auth, jobs, applications) and `dummyserver.js` as the bridge to the Python-based scoring engine — keeping the heavy ML dependency isolated from the main API.
